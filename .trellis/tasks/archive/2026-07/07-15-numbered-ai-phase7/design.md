# 阶段 7：安全监控与部署编号练习技术设计

## 目录与文件

- 目标根目录：`examples/ai/phase7/`。
- 目录映射以本任务 `prd.md` 的 Exercise Map 为唯一事实源。
- 每个目录至少有 `README.md` 和 `main.go`；`exercise.go` 保存本题专属接口、类型和函数签名，需要 fixture 或测试时增加最小必要文件。

## 骨架策略

默认离线或本地安全模式；任何凭证、真实发布、外部告警和破坏性故障注入都必须由学习者显式启用。

- 核心能力留在连续中文 TODO 中，入口、配置、错误出口和必要类型保持完整。
- 练习目录互不导入，公共概念通过重复的小接口说明，而不是新增跨练习公共包。
- 默认执行先校验占位配置；外部依赖创建必须位于校验之后。
- README 中明确后续深化内容，避免把多个路线项塞进同一个练习。

## 阶段特殊约束

部署练习生成安全模板和验证清单，不执行真实发布；Compose 配置不得包含真实密钥，故障注入只针对本地环境。

## 兼容性与回滚

- 本阶段只修改 `examples/ai/phase7/`、必要文档引用和本任务 Trellis 文件。
- 目录迁移或生成按练习独立检查；发现问题时只回滚对应目录，不影响其他阶段。
- 不覆盖工作区中无法归属的未提交文件；重叠内容先做逐文件对比。
