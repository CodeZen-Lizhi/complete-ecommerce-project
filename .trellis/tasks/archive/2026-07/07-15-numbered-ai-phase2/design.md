# 阶段 2：LLM API 与 Prompt 编号练习技术设计

## 目录与文件

- 目标根目录：`examples/ai/phase2/`。
- 目录映射以本任务 `prd.md` 的 Exercise Map 为唯一事实源。
- 每个目录至少有 `README.md`；需要运行代码时增加 `main.go`，需要 fixture 或测试时增加最小必要文件。

## 骨架策略

独立可运行 Go 骨架；默认占位配置必须在创建 Redis 或模型客户端前安全退出。

- 核心能力留在连续中文 TODO 中，入口、配置、错误出口和必要类型保持完整。
- 练习目录互不导入，公共概念通过重复的小接口说明，而不是新增跨练习公共包。
- 默认执行先校验占位配置；外部依赖创建必须位于校验之后。
- README 中明确后续深化内容，避免把多个路线项塞进同一个练习。

## 阶段特殊约束

前 6 个目录只迁移路径和更新引用，不重写内容；第 6 个未提交练习的 TODO 1–11、README 与安全退出行为必须完整保留。

## 兼容性与回滚

- 本阶段只修改 `examples/ai/phase2/`、必要文档引用和本任务 Trellis 文件。
- 目录迁移或生成按练习独立检查；发现问题时只回滚对应目录，不影响其他阶段。
- 不覆盖工作区中无法归属的未提交文件；重叠内容先做逐文件对比。
